2. 修复措施详情
================================================================

2.1 引入 Flask-WTF CSRFProtect 全局防护
----------------------------------------------------------------
文件: app.py:7-14

  [+] 新增导入: from flask_wtf.csrf import CSRFProtect
  [+] 新增配置: app.config['WTF_CSRF_ENABLED'] = True
  [+] 新增配置: app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'
  [+] 初始化: csrf = CSRFProtect(app)

作用:
  - CSRFProtect 自动拦截所有 POST/PUT/DELETE 请求
  - 验证请求中必须包含有效的 csrf_token
  - SESSION_COOKIE_SAMESITE = 'Lax' 限制跨站 Cookie 发送

2.2 添加 CSRF 错误处理
----------------------------------------------------------------
文件: app.py:17-21

  [+] 新增错误处理器 @app.errorhandler(400)
      自动识别 CSRF 验证失败，返回友好提示页面:
      "请求已过期或来源非法，请刷新页面后重试"
